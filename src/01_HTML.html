Hypertext Markup Language

  • Soporte sobre el que se sostiene una aplicación web.

Semántica

Las etiquetas HTML están dotadas de semántica, proporcionan significado a navegadores, motores de búsqueda y tecnologías asistenciales.

Tipos de contenido

Metadatos:
Contenido que prepara la presentación o comportamiento del resto de contenido y la relación con otros documentos.
Flujo:
Todo el contenido que se usa en el cuerpo del documento o aplicación, por cuerpo se refiere a todo lo que queda entre la etiqueta body.
Seccionado:
Contenido que define el ámbito de los encabezados y los pies, toda sección tiene potencialmente un encabezado y un esquema.
Encabezados:
Define el encabezado de una sección.
Redacción:
Texto del documento y los elementos que lo marcan dentro del párrafo.
Incorporado:
Contenido que se importa otro recurso de un vocabulario distinto.
Interactivo:
Contenido destinado a interactuar con el usuario.
Tangible:
Elementos cuyo modelo de contenido acepte flujo o redacción, que debería tener al menos un nodo y no esté oculto por el atributo "hidden".
Soporte para scripts:
No representan nada por sí mismos y son usados para contener scripts.

HTML5 Outliner

    Algoritmo por el cual los elementos de secciones definen un nuevo esquema, que potencialmente puede tener su propio header, footer y jerarquía de encabezados (h1 - h6). A día de hoy no se conoce ninguna implementación en ningún navegador gráfico o tecnología asistencial

  • https://developer.mozilla.org/en-US/docs/Web/Guide/HTML/Using_HTML_sections_and_outlines
  • https://gsnedders.html5.org/outliner/

WAI-ARIA - Web Accessibilty Initiative for Accessible Rich Internet Applications

  1. Si puedes usar un elemento nativo HTML con la semántica y el comportamiento que necesitas ya implementado, en vez de recondicionar otro elemento añadiendo ARIA role, estado o propiedad, hazlo.
  2. No cambies la semántica nativa
  3. Todos los controles interactivos ARIA deben ser usables a través del teclado
  4. No uses aria-hidden="true" o role="presentation" en un elemento enfocable o en un padre o ancestro, si usas las propiedades display:none o visibility:hidden no será enfocable
  5. Todos los elementos interactivos tienen que tener un nombre accesible

No usar ARIA es mejor que usarlo mal
